这个接口是 `baze平台` 的统一视频生成入口之一，适用于 MiniMax 文生视频能力。平台入口是 `/v1/videos`，请求会映射到 MiniMax 上游的 `POST https://api.minimax.io/v1/video_generation`。

## 快速开始

```bash
curl --request POST \
  --url https://baze-api.powerbuyin.top/v1/videos \
  --header "Authorization: Bearer <token>" \
  --header "Content-Type: application/json" \
  --data '{
    "model": "MiniMax-Hailuo-2.3",
    "prompt": "A woman walks through a neon street in the rain.",
    "seconds": "6",
    "size": "1080P",
    "metadata": {
      "prompt_optimizer": false
    }
  }'
```

## 字段映射与能力边界

| 平台字段 | MiniMax 上游字段 | 必填 | 说明 |
| --- | --- | --- | --- |
| `model` | `model` | 是 | 仅支持当前文档列出的文生视频模型 |
| `prompt` | `prompt` | 是 | 视频提示词 |
| `seconds` | `duration` | 否 | 平台会映射到上游时长字段 |
| `size` | `resolution` | 否 | 平台会映射到上游分辨率字段 |
| `metadata.*` | 同名扩展字段 | 否 | 透传上游扩展字段，如 `prompt_optimizer` |

当前支持的文生视频模型包括 `MiniMax-Hailuo-2.3`、`MiniMax-Hailuo-02`、`T2V-01-Director`、`T2V-01`。

## 常见错误

- 缺少 `prompt`
- 使用未支持的文生视频模型
- 在 `metadata` 中传入平台未公开支持的字段组合
