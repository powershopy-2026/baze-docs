This endpoint is one of the unified video generation entry points on the `baze platform` and targets the MiniMax text-to-video capability. The public path is `/v1/videos`, `seconds` maps to upstream `duration`, and `size` maps to upstream `resolution`.

## Quickstart

```bash
curl --request POST \
  --url https://baze-api.powerbuyin.top/v1/videos \
  --header "Authorization: Bearer <token>" \
  --header "Content-Type: application/json" \
  --data '{
    "model": "MiniMax-Hailuo-2.3",
    "prompt": "A woman walks through a neon street in the rain.",
    "seconds": "6",
    "size": "1080P",
    "metadata": {
      "prompt_optimizer": false
    }
  }'
```

## Field mapping and support boundaries

| Platform field | MiniMax upstream field | Required | Notes |
| --- | --- | --- | --- |
| `model` | `model` | Yes | Supported models are `MiniMax-Hailuo-2.3`, `MiniMax-Hailuo-02`, `T2V-01-Director`, `T2V-01` |
| `prompt` | `prompt` | Yes | Video prompt |
| `seconds` | `duration` | No | Requested video length |
| `size` | `resolution` | No | Requested output resolution |
| `metadata.*` | Matching extension fields | No | Supports public pass-through fields such as `prompt_optimizer` |

This page covers the current public field set on the platform, not the full upstream MiniMax capability set.

## Common errors

- Omitting `prompt`
- Using a model that is not supported for text to video
- Sending unsupported field combinations inside `metadata`
