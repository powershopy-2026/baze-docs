This endpoint is one of the unified video generation entry points on the `baze platform` and targets the MiniMax subject-reference-to-video capability. The public path is `/v1/videos`, and `subject_reference` is passed through using the upstream structure supported by the platform.

## Quickstart

```bash
curl --request POST \
  --url https://baze-api.powerbuyin.top/v1/videos \
  --header "Authorization: Bearer <token>" \
  --header "Content-Type: application/json" \
  --data '{
    "model": "S2V-01",
    "subject_reference": [
      {
        "type": "character",
        "image": [
          "https://example.com/assets/subject-reference.png"
        ]
      }
    ],
    "metadata": {
      "prompt_optimizer": false
    }
  }'
```

## Field mapping and support boundaries

| Platform field | MiniMax upstream field | Required | Notes |
| --- | --- | --- | --- |
| `model` | `model` | Yes | The currently supported model is `S2V-01` |
| `prompt` | `prompt` | No | Optional video prompt |
| `subject_reference` | `subject_reference` | Yes | Passed through with the supported upstream structure |
| `metadata.*` | Matching extension fields | No | Supports public pass-through fields such as `prompt_optimizer` and `callback_url` |

This page covers the subject-reference fields currently exposed by the platform, not the complete upstream MiniMax capability set.

## Common errors

- Omitting `subject_reference`
- Using a model other than `S2V-01`
- Sending a `subject_reference` structure outside the public support boundary
