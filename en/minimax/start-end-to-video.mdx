This endpoint is one of the unified video generation entry points on the `baze platform` and targets the MiniMax start-end-to-video capability. The public path is `/v1/videos`, `images[0]` maps to upstream `first_frame_image`, and `images[1]` maps to upstream `last_frame_image`.

## Quickstart

```bash
curl --request POST \
  --url https://baze-api.powerbuyin.top/v1/videos \
  --header "Authorization: Bearer <token>" \
  --header "Content-Type: application/json" \
  --data '{
    "model": "MiniMax-Hailuo-02",
    "images": [
      "https://example.com/assets/frame-start.png",
      "https://example.com/assets/frame-end.png"
    ],
    "prompt": "The child grows up naturally.",
    "seconds": "6",
    "size": "1080P"
  }'
```

## Field mapping and support boundaries

| Platform field | MiniMax upstream field | Required | Notes |
| --- | --- | --- | --- |
| `model` | `model` | Yes | The currently supported model is `MiniMax-Hailuo-02` |
| `images[0]` | `first_frame_image` | Yes | Start frame image |
| `images[1]` | `last_frame_image` | Yes | End frame image |
| `prompt` | `prompt` | No | Optional video prompt |
| `seconds` | `duration` | No | Requested video length |
| `size` | `resolution` | No | Requested output resolution |

`images` must contain exactly two items in `[start frame, end frame]` order.

## Common errors

- Sending fewer or more than 2 images
- Using a model other than `MiniMax-Hailuo-02`
- Reversing the image order and getting unexpected output
