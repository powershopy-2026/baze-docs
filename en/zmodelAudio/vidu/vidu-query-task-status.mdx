## Overview

Query the status and result of a Vidu audio generation task by task ID. This interface is specifically designed for audio tasks created via endpoints like `POST /vidu/v1/audio/text2audio` or `POST /vidu/v1/audio/timing2audio`.

## Usage

### Step 1: Create Audio Task

First, create an audio generation task using the appropriate endpoint (e.g., [Vidu Text-to-Audio](/en/zmodelAudio/vidu/vidu-text-to-audio) or [Vidu Timing-to-Audio](/en/zmodelAudio/vidu/vidu-timing-to-audio)). The response will include a `task_id`.

### Step 2: Query Task Status

Use the returned `task_id` to query the task status:

```bash
GET /vidu/v1/tasks/{task_id}
```

### Step 3: Get Results

When the task state is `success`, the response will contain the generated results in the `creations` array. Each creation includes:
- `id`: Creation ID
- `url`: Generated audio URL (valid for 24 hours)
- `cover_url`: Cover image URL (valid for 24 hours)

## Task States

The task goes through the following states:
- **created**: Task created successfully
- **queueing**: Task in queue
- **processing**: Task processing
- **success**: Generation successful (results available)
- **failed**: Task failed

## Response Structure

The response uses a unified structure with the `creations` array:

```json
{
  "state": "success",
  "err_code": "",
  "credits": 4,
  "payload": "",
  "creations": [
    {
      "id": "your_creations_id",
      "url": "your_generated_results_url",
      "cover_url": "your_generated_results_cover_url"
    }
  ]
}
```
